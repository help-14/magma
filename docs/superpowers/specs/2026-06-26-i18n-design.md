# i18n multilingual support for Magma

**Date:** 2026-06-26
**Status:** Approved

## Overview

Add Vietnamese + English i18n to the Magma self-hosted dashboard using `sveltekit-i18n`. Language is stored in `config/system.yaml` and selected via a Settings dropdown. English is the default.

## Architecture

```
sveltekit-i18n library
  ↓
src/lib/translations/
  index.js      ← i18n instance, exports { t, locale, loadTranslations }
  en.json       ← all English translation keys
  vi.json       ← all Vietnamese translation keys
  ↓
+layout.server.js  ← reads language from system config → data.locale
+layout.js         ← universal load, calls loadTranslations(data.locale, url.pathname)
  ↓
Components use {$t('key')} for all user-facing strings
```

### Key decisions

- **Flat/dot-notation keys** (e.g. `editor.save`, `weather.temperature`). Simple, no nesting beyond dots.
- **Language in system.yaml** — app-level setting, already managed by the config read/write pattern.
- **No URL-based routing** — single-language-per-user, config-driven.
- **Toast strings are translated** — they render via `{$t(...)}` in component code.
- **Widget titles and user data are NOT translated** — only the app UI.

### Config

`config/system.yaml` gets a new field at top level:

```yaml
version: 1
language: vi
system:
  dashboardGrid:
    cellWidth: 70
    cellHeight: 70
```

Default is `en`. The system config validator is updated to accept an optional `language` string.

### Data flow

1. `+layout.server.js` reads system config, extracts `language` (default `en`), passes as `data.locale`
2. New `+layout.js` (universal load) receives `data` from the server load, calls `await loadTranslations(data.locale, url.pathname)`
3. sveltekit-i18n lazy-loads the matching JSON locale file and initializes the `$t` store
4. All components render via `{$t('key')}`

### Settings UI

A `<select>` in the System tab sidebar (below bg image URL, or between cell inputs and bg URL):

```svelte
<Label class="grid gap-2 mt-4">
  <span class="text-magma-accent text-xs font-bold uppercase">Language</span>
  <select ...>
    <option value="en">English</option>
    <option value="vi">Tiếng Việt</option>
  </select>
</Label>
```

Changing language sets `language` in system YAML and immediately reloads translations. On save, the page reloads with the new language.

### WidgetTimer greeting locale fix

The `WidgetTimer.svelte` greeting strings are hardcoded in Vietnamese — they're extracted to translation keys (`timer.morning`, `timer.afternoon`, etc.) with proper English equivalents.

### Non-goals

- URL-based locale routing (`/en/`, `/vi/`)
- Browser auto-detection
- Pluralization/ICU message formats
- Runtime language switching without page reload
